Go

Why "Go"?

""Ogle" would be a good name for a Go debugger."

In a Nutshell

  • C-like, but modernized
  • Statically typed
  • Garbage collected
  • Concurrent
  • VERY fast compilation
  • Built-in testing
  • Built-in package management

History

  • Started at Google in 2007
  • Open sourced Nov 2009
  • 1.0 March 28, 2012
  • Currently at version 1.2.2
  • 1.3 beta released April 21, 2014

Syntax

C#

int i;
bool a, b, c;

int x = 1;
var y = 2;

Go

var i int
var a, b, c bool

var x, y int = 1, 2
var x, y = 1, 2
var different, types = true, "Works"

foo := "something"
foo, bar := true, 1.0

Collections

C#


int[] foo = new int[5];
foo[0] = 1;

var foo = new[] { 1, 2, 3, 4, 5 };


var foo = new List<int>{ 1, 2,3, 4, 5 };
foo.Add(6);
var bar = foo.ToList();

Go


var foo [5]int
foo[0] = 1

foo := [5]int{ 1, 2, 3, 4, 5 }
foo := [...]int{ 1, 2, 3, 4, 5 }

foo := []int{ 1, 2, 3, 4, 5 }
foo = append(foo, 6)
bar := make([]int, len(foo))
copy(bar, foo)

noCopy := foo[2:5] // [3 4 5]
noCopy := foo[2:] // [3 4 5 6]
